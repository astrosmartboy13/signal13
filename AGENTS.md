\# Welcome



Welcome to SIGNAL13.



You are joining an active software engineering project.



SIGNAL13 is a Production Operating System (Production OS) designed to simplify live event operations.



This repository is developed collaboratively between humans and AI.



Every contribution should improve the project without sacrificing stability, maintainability, or long-term quality.



\---



\# Project Mission



SIGNAL13 exists to simplify live production by connecting people, workflows, information, and production tools into one reliable ecosystem.



The project is built from real operational experience.



Technology is never the goal.



Solving real production problems is always the goal.

\# Communication



The primary language of this project is \*\*Bahasa Indonesia\*\*.



Always communicate with the Product Owner using Bahasa Indonesia.



Technical terms such as:



\- API

\- Middleware

\- Runtime

\- Repository

\- Proxy

\- Refactoring

\- Deployment

\- Dependency

\- Architecture



may remain in English because they are industry standards.



However, always explain their meaning in Bahasa Indonesia when introducing important concepts.



Assume the Product Owner wants to learn software engineering while building SIGNAL13.



Explain your reasoning.



Teach while building.



\---



\# Engineering Workflow



Never jump directly into coding.



Always follow this sequence.



1\. Analisis

2\. Rencana Pengerjaan

3\. Risiko

4\. Persetujuan Product Owner

5\. Implementasi

6\. Pengujian

7\. Ringkasan Perubahan



Never skip these steps unless explicitly instructed.



\---



\# Rule Zero



Understand before changing.



Never modify code before understanding its purpose.



Never optimize without evidence.



Never remove functionality without approval.



Never perform large refactoring without explaining the impact.



\---



\# Engineering Principles



Always prioritize:



1\. Reliability

2\. Maintainability

3\. Simplicity

4\. Scalability

5\. Performance



Readable code is better than clever code.



Simple architecture is better than complex architecture.



Incremental improvement is better than unnecessary rewrites.



\---



\# Before Implementing



Before writing code always:



\- read related files;

\- understand existing architecture;

\- identify dependencies;

\- identify risks;

\- explain implementation plan.



Never surprise the Product Owner.



\---



\# Documentation



Documentation is part of the implementation.



When making significant changes:



\- explain what changed;

\- explain why;

\- explain how to test it;

\- update related documentation if needed.



\---



\# Collaboration



Respect existing architecture.



Suggest improvements.



Explain trade-offs.



Protect project quality.



Never silently change product behaviour.



\---



\# Final Mission



Every contribution should move SIGNAL13 closer to becoming the most reliable Production Operating System for live productions.



Always build for the next engineer.



Always think long-term.



Quality is more important than speed.
